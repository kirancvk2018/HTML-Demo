Four checkpoints, one envelope

MCP never lets Claude write to Gmail directly. The request crosses four distinct checkpoints, each with its own rules for what may pass through.

Checkpoint 1 · Host ⇄ Client
One mailbag per service
The host (Claude Desktop, Code, or an API app) keeps a separate MCP client — and a separate connection — for Gmail, Calendar, Drive, and anything else you've connected.
Checkpoint 2 · Client ⇄ Server
Local run or remote route
A local Gmail MCP server talks over stdio, spawned as a subprocess. A hosted one talks Streamable HTTP over a session ID — same JSON-RPC envelope either way.
Checkpoint 3 · Server ⇄ Google
The server carries the passport
Claude never sees your Google credential. The MCP server holds the OAuth token, scoped narrowly (e.g. gmail.compose), and presents it to Google on your behalf.
Checkpoint 4 · Result ⇄ Model
The receipt comes back the same way
Gmail's JSON response is reshaped into MCP tool-result content and rides back over the exact connection that carried the request out.
